# 现有技术资产说明

> 本文档说明 V1.0 已有的可复用代码资产，帮助技术团队快速理解现有实现。  
> 源代码位于 `现有脚本参考/` 目录。

---

## 一、架构概览

```
当前架构（V1.0）：

  浏览器 HTML 面板
        │
        ▼  HTTP (port 9527)
  server.py（Python 桥接服务）
        │
        ▼  subprocess 调用
  scripts/*.py（各功能脚本）
        │
        ▼
  FFmpeg / YOLOv8 / LLM API / CMS API
```

所有脚本执行都通过 `server.py` 的 `/api/run` 端点，以命令行方式调用。前端通过 SSE 接收实时输出。

## 二、脚本清单 & 复用指南

### 2.1 copywrite_gen.py — 文案生成

**对应流水线**：Step 2

**调用方式**：
```bash
python3 copywrite_gen.py \
  --protagonist "主角名" \
  --event "事件描述" \
  --photos "3张自拍" \
  --video-desc "2段视频" \
  --style "反转打脸风" \
  --min-length 25 --max-length 30 \
  --kw-count 10 \
  --body-len 300 \
  --paragraphs 3 \
  --author "作者" \
  --category "今日吃瓜,网红黑料" \
  --save-dir "/path/to/output"
```

**输出**：标题 + 关键词 + 正文（stdout 或保存为 TXT）

**依赖**：
- LLM API（OpenAI 兼容格式 或 Anthropic API）
- 环境变量：`API_BASE`、`API_KEY`、`CW_MODEL`
- 提示词模板：`scripts/cw_prompts/` 目录下的 txt 文件

**复用建议**：核心生成逻辑可直接封装为后端 service 方法，去掉 CLI 参数解析即可。

### 2.2 image_rename.py — 图片重命名

**对应流水线**：Step 3

**调用方式**：
```bash
python3 image_rename.py \
  --folder "/path/to/images" \
  --prefix "重命名前缀" \
  --start 1 \
  --digits 2 \
  --separator "_"
```

**特性**：
- 冲突安全（先重命名为临时名再改为目标名）
- 自动跳过 `_cover` 文件
- 支持 `--dry-run` 预览模式

**依赖**：纯 Python，无第三方库

---

### 2.3 make_cover.py — 封面制作

**对应流水线**：Step 4

**调用方式**：
```bash
python3 make_cover.py \
  --folder "/path/to/images" \
  --layout triple \
  --head-margin 15 \
  --quality 95 \
  --output "/path/to/output"
```

**特性**：
- 支持 single / triple 拼接模式
- YOLOv8 人脸检测 → 智能裁剪，确保人脸在画面上方合适位置
- 输出到 `已处理/` 子目录

**依赖**：Pillow、numpy、ultralytics (YOLOv8)、yolov8n.pt 模型文件

### 2.4 image_watermark.py — 图片加水印

**对应流水线**：Step 5（图片水印部分）

**调用方式**：
```bash
python3 image_watermark.py \
  --folder "/path/to/images" \
  --watermark "watermark/logo.png" \
  --watermark-light "watermark/logo_light.png" \
  --position bottom-right \
  --wm-width 264 \
  --img-width 800 \
  --margin 10 \
  --opacity 100 \
  --output "/path/to/output"
```

**特性**：
- YOLOv8 人脸感知水印定位（自动避开人脸区域）
- 暗图自动切换浅色水印
- 6 种位置可选
- 支持 `--recursive` 子文件夹模式

**依赖**：Pillow、numpy、ultralytics (YOLOv8)

**V2.0 复用要点**：流水线中需要为 N 个平台分别加不同水印，即同一组图片调用 N 次，每次传不同的 `--watermark` 和 `--output`。

### 2.5 video_process.py — 视频处理（7 个子命令）

**对应流水线**：Step 5（视频水印部分）+ 工具箱

**子命令**：

| 子命令 | 功能 | 流水线/工具箱 |
|--------|------|--------------|
| `watermark` | 视频加水印（四角轮转/固定/双水印） | 流水线 Step 5 |
| `delogo` | 遮盖四角水印 | 工具箱 |
| `crop` | 裁掉四角水印 | 工具箱 |
| `blur-pad` | 虚化填充 | 工具箱 |
| `trim` | 裁掉片头片尾 | 工具箱 |
| `add-intro-outro` | 添加片头片尾 | 工具箱 |
| `concat` | 多视频合成 | 工具箱 |

**视频加水印调用方式**：
```bash
python3 video_process.py watermark \
  --input "/path/to/videos" \
  --watermark "watermark/brand.mov" \
  --mode corner-cycle \
  --scale-landscape 35 \
  --scale-portrait 35 \
  --resolution 720 \
  --codec h264_videotoolbox \
  --bitrate 2M \
  --output "/path/to/output"
```

**特性**：
- 自动检测最佳 GPU 编码器（macOS VideoToolbox / NVIDIA NVENC / AMD AMF / Intel QSV / CPU libx264）
- 支持压缩到指定大小

**依赖**：FFmpeg（外部二进制）

### 2.6 publish_api.py — CMS 发布

**对应流水线**：Step 6

**调用方式**：
```bash
# 登录
python3 publish_api.py --action login \
  --base-url "https://xxx/api.php" \
  --project-code "PROJECT" \
  --username "admin" \
  --password "pass"

# 上传视频
python3 publish_api.py --action upload-video \
  --folder "/path/to/task"

# 发布帖子（存草稿）
python3 publish_api.py --action publish \
  --folder "/path/to/task"
```

**特性**：
- AES-128-CBC 加密 + SHA256/MD5 签名（与 CMS 后台对接）
- 支持 R2 视频上传
- 支持图片上传 + 帖子排版组装 + 草稿存储

**依赖**：requests、pycryptodome

**V2.0 复用要点**：需要为每个平台维护独立的 session（不同 base-url + 凭据），并行调用。

### 2.7 server.py — HTTP 桥接服务

**V1.0 角色**：接收前端请求，执行脚本命令，流式返回输出。

**关键接口**：

| 路由 | 功能 |
|------|------|
| `POST /api/run` | 通用脚本执行器（allowlist 校验 + SSE 流式输出） |
| `POST /api/stop` | 终止正在运行的进程 |
| `POST /api/pick-folder` | 原生文件夹选择对话框 |
| `POST /api/save-file` | 保存文件到磁盘 |

**V2.0 改造建议**：
- 现有的通用 `/api/run` 模式不适合 V2.0 的任务管理需求
- 建议重构为按功能划分的 REST API（`/api/pipeline/create`、`/api/pipeline/{id}/step/{n}/confirm` 等）
- 脚本调用逻辑可保留，包装到后端 service 层
- 新增：用户认证、任务持久化（数据库）、WebSocket 实时推送
