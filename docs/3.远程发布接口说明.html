1. 基础说明

1.1 两套 Base URL

用途说明
项目列表使用固定入口 baseAxiod，默认 https://bpi5.ynrwkze.cc/api.php（代码中可从列表随机选一个）。
业务接口（登录、发布等）使用 createCustomAxios，baseURL 取自当前选中项目的 Project.api 数组中随机一条（按 project.type 缓存）。

必须先通过项目列表拿到项目并选中，后续请求的域名才是该项目配置的 API。

1.2 POST 请求体加密

除特别说明外，POST 请求 body 在发送前会经拦截器合并公共参数后整包加密（encryptData(JSON.stringify(requestData))），响应中 data 字段会解密后解析为 JSON。对接第三方时需与服务端约定同一套加解密规则。

1.3 公共请求参数（拦截器自动合并）

以下字段会与业务参数合并后一并加密（project_list 除外）：

字段说明
oauth_id每次请求生成
bundleId固定 com.pc.jyaw
version当前项目 Project.version，缺省 1.0.0
oauth_typeweb
languagezh
viaweb
token登录后的 auth_token；调用 /api/remote/project_list 时不带 token

2. 项目列表接口（用于拿全量项目 ID / 项目码做自动匹配）

2.1 接口

  • 方法 / 路径：POST /api/remote/project_list
  • 客户端：baseAxiod（固定 Base URL）
  • 认证：不传 token（拦截器中对 project_list 特殊处理）

2.2 响应数据模型（前端 Project）

字段类型说明
idnumber项目 ID，可做全量同步与自动匹配主键
titlestring展示名称
iconstring图标
apistring[]该项目业务 API 域名列表（登录、发布等走这里）
typestring项目类型标识
viastring如 sns 等，用于区分产品线
templatestring模板
videoboolean?是否视频相关能力等
versionstring接口版本，影响部分功能分支

2.3 项目码（与“自动匹配”的关系）

前端登录页所称的项目码并非接口直接返回字段，而是由 type 经 MD5 后取前 8 位得到：

项目码 = md5(project.type).slice(0, 8)

自动匹配流程建议：

  1. 调用 project_list 拉取全量项目；
  2. 用 id 与本地/运营侧配置映射，或同时缓存 id ↔ md5(type)[0:8]；
  3. 用户输入项目码时，在列表中查找 md5(type).slice(0,8) === 输入 的项目，选中后设置 currentProject，后续请求走该项目的 api Base URL。

3. 带项目码的登录

项目码只在前端用于选中对应 Project。选中后，根据项目类型走不同登录接口。

3.1 标准远程登录（非 HLW 类 type）

  • 路径：POST /api/remote/loginByPassword
  • 业务 body：{ username, password }
  • 成功：result.status !== 0 时，token 取 result.data（字符串），写入 auth_token

3.2 HLW 类项目（type 匹配 /hlw/）

  • 登录：POST /api/index/login，body：{ username, password }
  • 配置：POST /api/index/getClientConfig（登录流程中会拉取站点配置）
  • 成功：token 取 result.data.token

3.3 登录前配置（与发布页数据相关）

  • 非 HLW：POST /api/remote/config（loadAllConfig），用于分类、站点配置等。
  • HLW：POST /api/index/getClientConfig。

4. 发布类接口

以下均通过 createCustomAxios 请求当前项目的 api 域名。

4.1 视频上传（直传 + 登记）

典型流程（与前端 getR2uploadUrl + PUT + finalyUpload 一致）：

步骤接口 / 动作说明
1POST /api/remote/r2upload_info获取预签名上传信息；成功返回中含 uploadUrl、UploadName、publicUrl 等（字段名以实际解密后为准）。
2PUT 到返回的 uploadUrl请求体为视频二进制，Content-Type: video/mp4。
3POST /api/remote/upload_mv将视频登记到业务侧，前端示例参数：name、mp4_url、cover、upload_type: 1。

另：部分环境可能使用 S3 兼容地址直传（如 s3-asia.microservices.vip 获取链接），与上述 R2 流程二选一，以实际配置为准。

4.2 视频发布

  • 路径：POST /api/remote/create_video
  • 前端提交字段示例：
字段说明
title标题
cover封面
m3u8_uri选用视频（列表中的 m3u8 等）
m3u8_url可选，与所选视频相关
description描述
keywords关键词（提交前会将 #、多余逗号规范化）
tags标签（同上规范化）
cate_ids分类 ID 拼接字符串，如 1 或 1,2

成功判断：前端以 res.status === 1 为发布成功（与通用 status !== 0 的用法注意区分）。

4.3 帖子发布（标题、分类、标签等）

按项目分为两套：

A. 标准远程（create_update）

  • 路径：POST /api/remote/create_update
  • 主要字段：
字段说明
title标题
category_id分类，提交为逗号分隔字符串（如 1,2）
author_id作者 ID（可选，视配置是否展示）
tags / keyword标签、关键词（提交前规范化）
desc描述
cover封面
body正文（Markdown）
created创建时间字符串
sort / hot_num / top排序、热度、置顶等（视配置）
is_draft草稿 / 提交等状态（如 1 发布、3 草稿等，与版本有关）
is_best精选等
id编辑时传帖子 ID；新建可为 null
redirect管理员跳转链接（可选）

另：配置驱动的 products.inherit、products.constants 会合并进最终参数，以各产品 getProductParams() 为准。

B. HLW 线路（addArticle）

  • 路径：POST /api/index/addArticle
  • 字段示例：title、thumb（封面）、content（正文 Markdown）、tag、keyword
