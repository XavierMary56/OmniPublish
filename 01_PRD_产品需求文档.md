# OmniPublish V2.0 — 产品需求文档 (PRD)

> 版本：V2.0  
> 日期：2026-04-10  
> 状态：待开发  

---

## 一、产品概述

### 1.1 产品定位
OmniPublish V2.0 是面向公司内容编辑团队的**全链路发帖工作台**。将原本各自独立的文案生成、图片重命名、封面制作、加水印、上传发布等功能，串联为一条**自动推进的流水线**，并支持**一次操作分发到多个平台**。

### 1.2 核心改进（V1.0 → V2.0）

| 维度 | V1.0 | V2.0 |
|------|------|------|
| 工作方式 | 每个功能独立操作，手动跳转 | 流水线自动推进，人工只做确认 |
| 平台分发 | 一次只发一个平台 | 一次选多个平台，并行分发 |
| 任务追踪 | 无 | 任务看板 + 多平台子任务展开 |
| 数据统计 | 无 | 按平台/编辑/时间段统计 |
| 用户体系 | 无 | 每人独立账号登录，数据隔离 |
| 水印管理 | 每次手动选水印文件 | 业务线预配置，流水线自动匹配 |

### 1.3 使用角色

| 角色 | 说明 | 权限 |
|------|------|------|
| **编辑** | 日常发帖的一线操作人员 | 创建/管理自己的任务，查看自己的统计 |
| **组长** | 各组负责人 | 查看本组所有编辑的任务和统计 |
| **管理员** | 系统管理 | 全部权限：业务线管理、账号管理、全局统计 |

## 三、页面功能详述

### 3.1 工作台（Dashboard）

**目的**：登录后第一眼看到今日工作全貌。

**内容**：
- **统计卡片**：今日发帖 / 已完成 / 流水线进行中 / 发布失败 / 覆盖平台
- **最近流水线任务**（表格）：显示最近的任务列表，含流水线步骤进度条
- **各平台今日发帖**（柱状图）
- **待我处理**（仅当前登录用户）：
  - 需要我确认文案的任务
  - 需要我选择封面的任务
  - 等待切片完成的任务
  - 发布失败需重试的任务

> **权限规则**：  
> - 编辑：统计数据和任务列表只显示自己的  
> - 组长：显示本组数据  
> - 管理员：全局数据

---

### 3.2 发帖流水线（Pipeline） ← 核心功能

**目的**：一站式完成从素材到多平台发布的全流程。

#### 流水线 6 步：

```
Step 1          Step 2          Step 3          Step 4          Step 5          Step 6
素材&选平台 ──→ 文案生成 ──→ 图片重命名 ──→ 封面制作 ──→ 水印处理 ──→ 上传&发布
                 ↑人工确认      自动推进        ↑人工确认      自动处理       ↑人工触发
```

#### Step 1：素材 & 平台选择
- **输入**：
  - 素材文件夹路径（拖入或手动填写）
  - 选择目标发布平台（多选，从 31 个业务线中勾选）
- **自动行为**：
  - 扫描文件夹，识别图片/视频/TXT 文件数量并展示
  - 选择平台后，显示已选数量
- **推进条件**：已选文件夹 + 至少选 1 个平台

#### Step 2：文案生成
- **输入**：主角、事件、生活照描述、视频内容描述、文风、作者、分类（可选）、字数参数
- **分类字段逻辑**：
  - 分类选项**不是固定的**，而是根据 Step 1 选择的目标平台动态加载
  - 每个业务线在「业务线管理」中维护自己的分类库（后台导入）
  - 如果目标平台有分类库 → 显示分类下拉/多选，编辑必选
  - 如果目标平台无分类库 → 隐藏分类字段，不参与文案生成
  - 如果选了多个平台且分类库不同 → 取并集展示，发布时各平台只用自己支持的分类
- **自动行为**：调用 AI 生成标题 + 关键词 + 正文
- **人工操作**：
  - 阅读生成结果
  - 可直接编辑修改标题/关键词/正文
  - 不满意可「重新生成」
  - 满意后点「确认文案，下一步」
- **推进条件**：用户点击确认按钮
- **自动联动**：确认后，标题自动填入 Step 3 的重命名前缀

#### Step 3：图片重命名
- **输入**：重命名前缀（自动从标题提取，可修改）、起始编号、分隔符
- **自动行为**：实时预览重命名结果（旧文件名 → 新文件名）
- **人工操作**：确认前缀是否满意
- **推进条件**：用户点击确认 → 自动执行重命名 → 完成后自动进入 Step 4

#### Step 4：封面制作
- **输入**：拼接方式（单图/双拼/三拼）、封面尺寸、头顶留白
- **自动行为**：从素材图片中自动生成 2~3 个候选封面（人脸检测智能裁剪）
- **人工操作**：
  - 从候选封面中选一个
  - 不满意可「重新生成」或「手动选图」
  - 满意后点确认
- **推进条件**：用户选中一个封面并确认

#### Step 5：水印处理
- **自动行为**：
  - 根据 Step 1 选择的平台，读取各平台的水印配置
  - 展示每个平台的水印方案（水印文件、位置、缩放）
  - 显示预估处理时间
- **人工操作**：
  - 检查各平台水印方案是否正确
  - 可修改单个平台的水印设置
  - 确认后系统自动处理
- **自动处理**：
  - 为每个目标平台生成独立的带水印图片集
  - 为每个目标平台生成独立的带水印视频
- **推进条件**：所有平台的水印处理完成 → 自动进入 Step 6

#### Step 6：上传 & 发布
- **自动行为**：
  - 各平台并行上传视频
  - 实时显示每个平台的上传进度/切片进度
- **展示**：每个目标平台一张卡片，各自显示：
  - 上传进度条
  - 切片状态（等待/进行中/完成/失败）
  - 操作按钮（切片完成后激活「存草稿」按钮）
- **人工操作**：
  - 单个平台：切片完成后点「存草稿」
  - 批量操作：「一键发布所有已就绪平台」
- **完成条件**：所有平台发布成功

#### 流水线异常处理

| 异常 | 处理方式 |
|------|---------|
| AI 文案生成失败 | 显示错误信息，可重试 |
| 封面生成失败 | 回退到手动选图模式 |
| 单平台水印处理失败 | 标记失败，不阻塞其他平台 |
| 上传超时 | 显示重试按钮 |
| 切片失败 | 显示失败原因 + 重试 |
| 发布 API 调用失败 | 显示错误信息 + 重试 |

### 3.3 任务看板（Tasks）

**目的**：追踪所有发帖任务的状态。

**功能**：
- **筛选栏**：全部 / 进行中 / 待确认 / 切片中 / 已完成 / 失败
- **统计卡片**：各状态数量
- **任务表格**：
  - 列：任务 ID、标题、目标平台 badges、流水线进度（step pills）、状态、编辑、时间、操作
  - **行展开**：点击任务行可展开，显示该任务在各平台的独立分发状态
    - 同一个任务发 5 个平台 → 展开后显示 5 行子任务
    - 每个子任务有独立的流水线进度

**任务状态定义**：

| 状态 | 含义 | 颜色 |
|------|------|------|
| 进行中 | 流水线自动执行中 | 蓝色 |
| 待确认 | 等待人工确认（文案/封面） | 橙色 |
| 切片中 | 视频已上传，等待平台切片完成 | 橙色 |
| 已完成 | 所有平台发布成功 | 绿色 |
| 部分完成 | 部分平台成功，部分失败 | 绿色+红色 |
| 失败 | 发布失败 | 红色 |

### 3.4 数据统计（Analytics）

**时间维度**：今日 / 本周 / 本月 / 自定义

**统计内容**：
- **总览卡片**：总发帖量、成功率、平均流水线耗时、人工介入率
- **各平台发帖统计表**：平台名、发帖数、成功/失败数、成功率、平均耗时
- **各编辑发帖排名**
- **流水线各步骤平均耗时分析**（柱状图）

**权限**：
- 编辑：只看自己的数据
- 组长：看本组数据
- 管理员：看全局数据

### 3.5 工具箱（Toolbox）

**目的**：保留 V1.0 的独立工具，用于流水线外的特殊需求。

**工具列表**（不在流水线内，独立使用）：
- 遮盖四角水印
- 裁掉四角水印
- 虚化填充
- 片头片尾处理
- 视频合成
- 视频压缩
- 智能封面（AI 从视频选帧）
- 图片格式转换
- 独立文案生成
- 评论审核

---

### 3.6 业务线管理（Platforms）

**目的**：配置 35 个业务线的参数，流水线自动读取。

**每个业务线的配置项**：
- 基本信息：名称、所属部组
- 分类库：该平台的内容分类列表（可为空，为空表示该平台不需要分类；支持批量导入）
- 图片水印：水印文件（**本地上传 PNG**）、位置、宽度、透明度
- 视频水印：水印文件（**本地上传 MOV/PNG**）、模式（四角轮转/固定/双水印）、缩放比例
- 发布配置：API 入口 URL、项目代码
- 排版模板：帖子内容排版规则

**水印文件管理**：
- 水印文件通过拖拽或点击上传，存储到服务端统一目录
- 上传后显示缩略图预览
- 支持替换（重新上传覆盖旧文件）

**操作**：新增 / 编辑 / 删除

---

### 3.7 账号管理（Accounts）

**目的**：管理各平台的后台登录凭据。

**注意**：这里管的是**发布目标平台的后台账号**（用于调 API 发帖），不是编辑自己的登录账号。

**展示**：平台名、入口 URL、用户名、登录状态、上次登录时间  
**操作**：重新登录 / 编辑凭据

## 四、用户体系

### 4.1 编辑登录

- 每个编辑用自己的账号登录工作台
- 登录后顶栏和侧栏底部显示：姓名 + 部组
- 所有页面的数据默认按当前登录用户过滤

### 4.2 数据隔离规则

| 数据 | 编辑可见范围 | 组长可见范围 | 管理员可见范围 |
|------|-------------|-------------|---------------|
| 任务列表 | 仅自己的 | 本组所有 | 全部 |
| 统计数据 | 仅自己的 | 本组汇总 | 全局汇总 |
| 待我处理 | 仅自己的 | 仅自己的 | 仅自己的 |
| 业务线管理 | 只读 | 只读 | 读写 |
| 账号管理 | 不可见 | 不可见 | 可见 |

## 五、非功能需求

### 5.1 技术约束
- 前端：单页应用（SPA），暗色主题，保持与原型一致的视觉风格
- 后端：需从现有的单一 `/api/run` 命令执行模式，重构为正规 REST API
- 现有 Python 脚本可作为后端处理模块直接复用
- 视频处理依赖 FFmpeg，封面/水印依赖 YOLOv8 + Pillow

### 5.2 性能要求
- 流水线 Step 1~4 交互响应 < 2s
- 水印处理：支持同时处理 5 个平台的水印任务（并行）
- 多平台上传：支持至少 3 路并行上传

### 5.3 部署方式
- 本地部署（编辑机器上运行）或内网服务器部署
- 需要访问各平台后台 API（外网）
