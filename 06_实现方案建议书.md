## 一、核心决策：技术栈选型

### 1.1 选型分析

| 方案 | 前端 | 后端 | 数据库 | 任务队列 | 实时推送 |
|------|------|------|--------|---------|---------|
| **A: Python 全栈** | Vue 3 SPA | FastAPI | SQLite/PostgreSQL | Celery+Redis | WebSocket |
| **B: PHP + Python 混合** | Vue 3 SPA | PHP Yaf（API层）+ Python（脚本层） | MySQL | PHP Swoole | SSE |
| **C: Python 全栈轻量** | Vue 3 SPA | FastAPI | SQLite | 内置 asyncio | SSE |

### 1.2 推荐方案：C（Python 全栈轻量）

**理由**：

1. **现有脚本全部是 Python**：7 个核心脚本（copywrite_gen、image_rename、make_cover、image_watermark、video_process、publish_api、server.py）全部是 Python，直接 import 调用比 subprocess 更高效、更易错误处理
2. **部署简单**：SQLite 无需额外数据库服务，单机部署一个 Python 进程即可
3. **团队成本**：不需要 PHP 和 Python 两套人员维护
4. **server.py 已有基础**：V1.0 的 server.py 已经是 Python HTTP 服务，升级为 FastAPI 改动最小
5. **异步并发**：FastAPI + asyncio 原生支持并行水印处理、并行上传，无需额外 Celery

**如果团队强烈倾向 PHP**（方案 B），可以在 PHP Yaf 层做 API 网关 + 用户认证 + 数据库操作，Python 脚本通过 `exec()` 或 HTTP 内部调用执行。但这增加了架构复杂度，不推荐。

## 三、数据库设计

### 3.1 表结构（SQLite，后续可迁移 PostgreSQL）

```sql
-- 用户表
CREATE TABLE users (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    username    TEXT NOT NULL UNIQUE,

    -- 分类库
    categories  TEXT DEFAULT '[]',      -- JSON array: ["今日吃瓜","网红黑料"]
    -- 图片水印配置
    img_wm_file     TEXT DEFAULT '',    -- 水印文件路径（服务端相对路径）
    img_wm_position TEXT DEFAULT 'bottom-right',
    img_wm_width    INTEGER DEFAULT 264,
    img_wm_opacity  INTEGER DEFAULT 100,
    -- 视频水印配置
    vid_wm_file     TEXT DEFAULT '',
    vid_wm_mode     TEXT DEFAULT 'corner-cycle', -- corner-cycle/fixed/diagonal
    vid_wm_scale    INTEGER DEFAULT 35,
    vid_wm_file2    TEXT DEFAULT '',    -- 双水印模式第二个文件
    -- 发布配置
    api_base_url    TEXT DEFAULT '',
    project_code    TEXT DEFAULT '',
    -- 排版模板
    layout_template TEXT DEFAULT '',
    -- 账号凭据（加密存储）
    cms_username    TEXT DEFAULT '',
    cms_password    TEXT DEFAULT '',     -- AES 加密
    session_token   TEXT DEFAULT '',
    session_expires TEXT DEFAULT '',
    --
    is_active   INTEGER DEFAULT 1,
    created_at  TEXT DEFAULT (datetime('now')),
    updated_at  TEXT DEFAULT (datetime('now'))
);

-- 发帖任务主表
CREATE TABLE tasks (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    task_no         TEXT NOT NULL UNIQUE, -- "#0053" 格式的可读编号
    title           TEXT DEFAULT '',      -- Step 2 确认后填入
    folder_path     TEXT NOT NULL,
    created_by      INTEGER NOT NULL REFERENCES users(id),
    current_step    INTEGER DEFAULT 0,   -- 0~5
    status          TEXT DEFAULT 'draft', -- draft/running/awaiting_confirm/
                                          -- slicing/done/partial/failed
    -- Step 1 数据
    target_platforms TEXT DEFAULT '[]',   -- JSON: [platform_id, ...]
    file_manifest   TEXT DEFAULT '{}',   -- JSON: {images:[], videos:[], txts:[]}
    -- Step 2 数据
    protagonist     TEXT DEFAULT '',
    event_desc      TEXT DEFAULT '',
    style           TEXT DEFAULT '反转打脸风',
    author          TEXT DEFAULT '',
    categories      TEXT DEFAULT '[]',   -- JSON array
    confirmed_title TEXT DEFAULT '',
    confirmed_keywords TEXT DEFAULT '',
    confirmed_body  TEXT DEFAULT '',
    -- Step 3 数据
    rename_prefix   TEXT DEFAULT '',
    rename_mapping  TEXT DEFAULT '[]',   -- JSON: [{old, new}]
    -- Step 4 数据
    cover_layout    TEXT DEFAULT 'triple',
    cover_path      TEXT DEFAULT '',
    cover_candidates TEXT DEFAULT '[]',  -- JSON: [path1, path2, path3]
    -- 时间
    created_at      TEXT DEFAULT (datetime('now')),
    updated_at      TEXT DEFAULT (datetime('now')),
    finished_at     TEXT DEFAULT NULL
);

-- 步骤状态表（每个任务最多 6 条记录）
CREATE TABLE task_steps (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id     INTEGER NOT NULL REFERENCES tasks(id),
    step        INTEGER NOT NULL,       -- 0~5
    status      TEXT DEFAULT 'pending', -- pending/running/awaiting_confirm/done/failed/skipped
    started_at  TEXT DEFAULT NULL,
    finished_at TEXT DEFAULT NULL,
    data        TEXT DEFAULT '{}',      -- JSON: 各步骤的输入输出
    error       TEXT DEFAULT NULL,
    UNIQUE(task_id, step)
);

-- 平台子任务表（Step 5/6 按平台拆分）
CREATE TABLE platform_tasks (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id         INTEGER NOT NULL REFERENCES tasks(id),
    platform_id     INTEGER NOT NULL REFERENCES platforms(id),
    -- Step 5: 水印
    wm_status       TEXT DEFAULT 'pending',  -- pending/running/done/failed
    wm_progress     REAL DEFAULT 0,
    wm_images_dir   TEXT DEFAULT '',
    wm_video_path   TEXT DEFAULT '',
    wm_cover_path   TEXT DEFAULT '',
    wm_error        TEXT DEFAULT NULL,
    -- Step 6: 上传
    upload_status   TEXT DEFAULT 'pending',  -- pending/uploading/done/failed
    upload_progress REAL DEFAULT 0,
    mp4_url         TEXT DEFAULT '',
    -- Step 6: 切片
    transcode_status TEXT DEFAULT 'pending', -- pending/transcoding/done/failed
    transcode_progress REAL DEFAULT 0,
    video_url       TEXT DEFAULT '',         -- m3u8 URL
    -- Step 6: 发布
    publish_status  TEXT DEFAULT 'pending',  -- pending/publishing/published/failed
    publish_result  TEXT DEFAULT '{}',       -- JSON: API 返回
    publish_error   TEXT DEFAULT NULL,
    --
    created_at      TEXT DEFAULT (datetime('now')),
    updated_at      TEXT DEFAULT (datetime('now')),
    UNIQUE(task_id, platform_id)
);

-- 操作日志（可选，用于审计）
CREATE TABLE task_logs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id     INTEGER NOT NULL REFERENCES tasks(id),
    step        INTEGER DEFAULT NULL,
    platform_id INTEGER DEFAULT NULL,
    level       TEXT DEFAULT 'info',    -- info/warn/error
    message     TEXT NOT NULL,
    created_at  TEXT DEFAULT (datetime('now'))
);

-- 索引
CREATE INDEX idx_tasks_created_by ON tasks(created_by);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_created_at ON tasks(created_at);
CREATE INDEX idx_platform_tasks_task ON platform_tasks(task_id);
CREATE INDEX idx_task_logs_task ON task_logs(task_id);
```

## 五、核心流程：脚本集成方式

### 5.1 现有脚本 → Service 层封装

**原则**：不再通过 subprocess 调用脚本，而是 **import 脚本中的核心函数直接调用**。

```python
# services/copywrite_service.py
from scripts.copywrite_gen import build_system_prompt, build_user_prompt, call_api, parse_result

class CopywriteService:
    async def generate(self, task_id, params):
        """异步生成文案，通过 WebSocket 推送进度。"""
        # 更新 step 状态
        await self.db.update_step(task_id, step=1, status='running')

        system_prompt = build_system_prompt(PROMPTS_DIR, params['style'])
        user_prompt = build_user_prompt(params)

        # 在线程池中执行（因为 urllib 是同步的）
        result_text = await asyncio.to_thread(
            call_api, system_prompt, user_prompt, API_BASE, API_KEY, MODEL
        )

        result = parse_result(result_text, params['author'], params['category'])

        # 更新数据库
        await self.db.update_step(task_id, step=1, status='awaiting_confirm', data=result)

        # WebSocket 推送
        await self.ws.send(task_id, {'type': 'step_changed', 'step': 1, 'status': 'awaiting_confirm'})

        return result
```

```python
# services/watermark_service.py
from scripts.image_watermark import process_folder
from scripts.video_process import _process_single_watermark

class WatermarkService:
    async def process_all_platforms(self, task_id, platforms):
        """并行处理所有平台的水印。"""
        await self.db.update_step(task_id, step=4, status='running')

        # 为每个平台创建异步任务
        tasks = []
        for platform in platforms:
            task = asyncio.create_task(
                self._process_single_platform(task_id, platform)
            )
            tasks.append(task)

        # 并行执行，单个失败不影响其他
        results = await asyncio.gather(*tasks, return_exceptions=True)

        # 检查是否全部完成
        all_done = all(not isinstance(r, Exception) for r in results)
        status = 'done' if all_done else 'partial'
        await self.db.update_step(task_id, step=4, status=status)

    async def _process_single_platform(self, task_id, platform):
        """单个平台的水印处理（在线程池执行）。"""
        await self.db.update_platform_task(task_id, platform['id'], wm_status='running')

        try:
            # 图片水印
            await asyncio.to_thread(
                process_folder,
                folder=task.folder_path,
                output=f"{task.folder_path}/wm_{platform['name']}",
                watermark_path=platform['img_wm_file'],
                position=platform['img_wm_position'],
                wm_width=platform['img_wm_width'],
                opacity=platform['img_wm_opacity'],
            )

            # 视频水印（如有视频文件）
            # ... 类似调用 video_process

            await self.db.update_platform_task(task_id, platform['id'], wm_status='done')
            await self.ws.send(task_id, {
                'type': 'wm_progress',
                'platform_id': platform['id'],
                'progress': 100
            })
        except Exception as e:
            await self.db.update_platform_task(
                task_id, platform['id'],
                wm_status='failed', wm_error=str(e)
            )
```

### 5.2 长耗时操作处理策略

| 操作 | 耗时 | 处理方式 |
|------|------|---------|
| AI 文案生成 | 5-15s | `asyncio.to_thread` + SSE 流式 |
| 图片重命名 | <1s | 同步执行 |
| 封面生成 | 3-10s | `asyncio.to_thread` |
| 图片水印（N 平台） | 1-5min | `asyncio.gather` 并行 + WebSocket 进度 |
| 视频水印（N 平台） | 3-15min | `asyncio.gather` 并行 + WebSocket 进度 |
| 视频上传 | 5-30min | `aiohttp` 异步上传 + 进度回调 |
| 切片等待 | 1-10min | 异步轮询 + WebSocket 推送 |

## 六、前端建议

### 6.1 技术选型

```
Vue 3 + TypeScript + Vite
├── Pinia                 状态管理
├── Vue Router            路由（SPA 7 个页面）
├── Axios                 HTTP 客户端
├── native WebSocket      实时推送
├── Element Plus 或 Naive UI   组件库（暗色主题）
└── ECharts               图表
```

### 6.2 为什么不用 React

- 原型 HTML 已经是 Vue 风格的模板驱动（双花括号、v-for 等更直觉）
- Vue 3 的 Composition API 对这种中等复杂度的表单+看板应用更简洁
- Element Plus / Naive UI 暗色主题开箱即用

### 6.3 关键前端状态

## 九、关键风险和应对

| 风险 | 影响 | 应对 |
|------|------|------|
| **35 个平台 API 差异大** | 发布逻辑需要大量适配 | publish_api.py 已处理 hlw/标准两种类型，逐步扩展 |
| **视频水印处理耗时长** | 5 个平台 × 2 个视频 = 10 次 FFmpeg | 并行处理 + 进度推送，用户可离开页面 |
| **YOLOv8 模型首次下载** | 几百 MB，首次启动慢 | Docker 镜像预装模型；或启动时后台下载 |
| **LLM API 不稳定** | 文案生成失败 | 已有 3 次重试；加 fallback 到另一个模型 |
| **SQLite 并发写** | 多用户同时操作 | WAL 模式 + 写入序列化；后期迁移 PostgreSQL |
| **大文件上传断点续传** | 2GB+ 视频上传可能中断 | 分片上传 + 已完成分片记录 |

## 十、部署方案

### 10.1 开发环境

```bash
# 后端
cd backend
pip install fastapi uvicorn[standard] aiosqlite python-jose[cryptography] passlib[bcrypt]
uvicorn main:app --reload --port 9527

# 前端
cd frontend
npm install
npm run dev -- --port 3000
```

### 10.2 生产环境（Docker）

```yaml
# docker-compose.yml
services:
  omnipub:
    build: .
    ports:
      - "9527:9527"
    volumes:
      - ./data:/app/data          # SQLite + 上传文件
      - /path/to/素材:/materials   # 素材目录挂载
    environment:
      - OPENAI_API_KEY=sk-xxx
      - OMNIPUB_AUTH_SECRET=xxx
```

### 10.3 与现有 Docker 环境集成

如果需要在现有的 `docker-php7.3` 环境中运行：
- 新增一个 Python 服务容器（独立于 `fpm-server`）
- 通过 Docker network 共享文件系统
- Nginx 反向代理 `/omnipub/` 到 Python 服务

## 十一、总结

| 决策项 | 推荐 | 理由 |
|--------|------|------|
| 后端 | **FastAPI (Python)** | 脚本直接 import，异步原生支持 |
| 前端 | **Vue 3 + Pinia** | 表单驱动型 UI 更简洁 |
| 数据库 | **SQLite (WAL)** | 单机部署，无额外依赖 |
| 实时推送 | **WebSocket** | 双向通信，比 SSE 更灵活 |
| 任务队列 | **asyncio 内置** | 无需 Celery/Redis，够用 |
| 部署 | **Docker 单容器** | 简单，可后期拆分 |

这个方案的核心思路是：**最大化复用现有 Python 脚本，最小化架构复杂度，一步步从 MVP 迭代到完整功能**。
